# Instant Collaboration Using Mural or Google Docs

This method provides a structured collaboration format for teams to store work in a shared repository. This allows the prototype makers to access copy and content to implement in the prototype. It eliminates the need to ask, “Did you update that?” or “Can you share that with me when you finish?” It keeps communication quick and asynchronous, and transparent communication and asset storage are readily available after the sprint.

* **Time:** Variable
* **Activity:** Group
* **Output:** Shared workspace for prototype content and assets.

### Directions

1. Translate the jobs to Mural or Google Docs as a Kanban board.
2. Create sections for each screen of the prototype.
3. Team members place assets into the appropriate section.
4. The stitcher gathers and organizes all assets to make sure the board is accurate and ready for the makers to pull from.
5. Prototype makers copy and paste assets into the prototype.
6. Comment and annotate to communicate on edits or any additional content needed.
