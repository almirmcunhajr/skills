# Assign Tasks

During the Prototype phase, it helps to divide and conquer. Assign roles to each person on the team (maker, writer, stitcher, reviewer, interviewer, etc.) and make sure everyone is clear on what they’re doing. For example, the researcher and project manager could recruit test participants while the designer creates assets and the engineer builds the prototype.

* **Time:** 30 mins
* **Activity:** Group
* **Output:** Role/task ownership.
* **Source:** Core Method

### Directions

1. List all the tasks that need to be completed. For example: mock making, writing, stitching, reviewing, scriptwriting for the interview, etc.
2. Allocate each task to a Sprint participant or form groups of two or three people.
3. Keep everyone engaged at this time to keep people from disappearing.
4. Schedule user validation sessions.
